# Local Task Management Guide

## Overview
Agents should use simple markdown files within the `.agent/tasks/` directory for tracking progress and plans.

## Session Files

- **`plan.md`**: Primary plan documentation file generated by PM agent.
- **`session-{flow}.md`**: Session tracking file.
- **`progress-{agent-id}.md`**: Step-by-step progress logging by the working agent.
- **`result-{agent}.md`**: Final completion / QA report.

## Rules
1. Never overwrite another agent's file.
2. Read `task-board.md` for current assignments, but do NOT edit it directly unless you manage the board.
3. Update `progress-{agent-id}.md` every ~3-5 steps so state is not lost.
